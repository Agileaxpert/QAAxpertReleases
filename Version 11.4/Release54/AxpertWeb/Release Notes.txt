TSK-0368 -- Enhancement: Added an additional option to set Home / Landing page as HTML template by using the below mentioned developer option(Landing Structure). Currently home / Landing page can be set as selected tstruct / iview / html pages. In addition main page templates can also be customized to show the data in cards view as home / landing page.
   1. Developer option :
       Property: Landing Structure
       Property Code: General
       Property Value: tstruct / iview / html pages / html templates
       Form/Report: Reports / Tstructs
       User Role: All / Single role  
    Note: a. Property value select as 'tstruct' should choose the 'form' in Form/Report dropdown.
          b. Property value select as 'iview' should choose the 'iview' in Form/Report dropdown.
          c. Property value select as 'html pages' should choose the 'html page' in Form/Report dropdown.
          d. If Property value select as 'html templates', than 'Landing Page HTML Template' file name should be entered in 'Property Info' and Form/Report dropdown will be empty.
   2. Plugin Custom Code Page: A new option named "Landing Page HTML Template" has been added to the Plugin Custom Code menu. By selecting "Add Landing Page HTML Template" from the Plugin Custom Code button menu, users can create a new HTML template file (The generated file is saved in the LandingPageHTMLTemplates folder located under the AxpertWeb directory of the project (i.e., AxpertWeb\<ProjectName>\LandingPageHTMLTemplates)).
   3.  Template Name Mapping: The Landing Page HTML Template file name must exactly match the value entered in the 'Property Info' field when Property Value is set to 'html templates'.

TKT-0422 -QA- Currently, records are pushed to the queue using the CachedSave API. The processing result is communicated asynchronously through FCM notifications.
In some scenarios, FCM notifications may not be received or triggered successfully. In such cases, there is no mechanism for mobile to retrieve the processing status of the submitted queue records. To address this gap, a new API is required to fetch the status of one or more CachedSave requests based on their axm_queueids.
Note to QA: The transaction result is stored in the axactivemessages table. Based on axm_queueids, can call the GetSQL API and display the data in the mobile reports.
